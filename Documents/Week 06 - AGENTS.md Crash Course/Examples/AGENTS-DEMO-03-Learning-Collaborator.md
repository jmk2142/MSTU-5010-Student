# AGENTS-DEMO-03-Learning-Collaborator.md

This is a teaching demo file, not a live repo instruction file.
Use this filename for reference examples so tools do not treat it as active `AGENTS.md` policy.

## Purpose
Show an advanced policy where the agent is both a builder and a learning collaborator.

## Demo policy
- Build rule:
  - Complete the requested edit with minimal file changes.
- Reflection rule:
  - After edits, append a 4-line learning log:
    - what changed
    - what failed or was uncertain
    - what principle you used
    - what to try next time
- Commit-quality rule:
  - Propose a commit message in this format:
    - `type(scope): change`
    - `Boundary rationale: ...`
- Coaching rule:
  - End with one coaching question that helps the human improve the design.
- Safety rule:
  - If instructions conflict, stop and ask for clarification.

## Example class task
Refactor a small section for clarity, then produce the learning log, commit message suggestion, and one coaching question.
